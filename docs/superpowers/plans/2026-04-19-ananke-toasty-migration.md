# Ananke Toasty Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate Ananke's database layer from hand-rolled rusqlite to `toasty = "0.4"` with the `sqlite` feature, per the parent spec's original intent. Keep a tiny rusqlite side-channel only for pragmas toasty cannot issue (`auto_vacuum`, `incremental_vacuum`).

**Architecture:** `Database` wraps `toasty::Db`. Six models (`Service`, `ServiceConfigVersion`, `RunningService`, `ServiceLog`, `AllocationEvent`, `Oneshot`) defined via `#[derive(toasty::Model)]` with `#[key(partition, local)]` for composite keys. Initial migration generated by `toasty migrate generate` and checked in under `toasty/migrations/`. All ~15 call-site query paths move to toasty's builder DSL in one pass; no hybrid rusqlite/toasty coexistence inside a single surface.

**Tech Stack:** Rust 2024, `toasty = "0.4"` (SQLite feature), `rusqlite = "0.32"` (pragma side-channel only), existing phase 1-4 async infrastructure.

**Parent spec:** `docs/superpowers/specs/2026-04-19-ananke-toasty-migration.md`.

---

## Revert criteria

Stop and restore rusqlite if:
- Toasty's `Vec<CreateStatement>` batch takes > 5 ms for a 100-row insert (measure in Task 6).
- Toasty's query DSL demonstrably lacks a primitive we need (see Task 9 for the retention-trim subquery — that's the trickiest one).
- The extended smoke fails end-to-end.

Before reverting, amend `docs/spec.md` §12 to document the decision.

## Task 3: Database type rewrite

- [ ] **Step 1: Create the pragma side-channel**

Create `src/db/pragma.rs`:

```rust
//! Thin rusqlite side-channel for pragmas toasty cannot issue.
//!
//! Only two pragmas live here:
//! - `PRAGMA auto_vacuum = INCREMENTAL` — run once on a fresh database
//!   file before any CREATE TABLE. auto_vacuum is file-persistent; a
//!   new toasty connection pool will inherit it.
//! - `PRAGMA incremental_vacuum(N)` — run by retention to reclaim pages.
//!   Issued against a fresh rusqlite connection each time; cheap.

use std::path::Path;

use rusqlite::Connection;

pub fn set_auto_vacuum_incremental(path: &Path) -> rusqlite::Result<()> {
    let conn = Connection::open(path)?;
    // auto_vacuum only takes effect on an empty DB. No-op if tables exist.
    conn.execute_batch("PRAGMA auto_vacuum = INCREMENTAL")?;
    Ok(())
}

pub fn incremental_vacuum(path: &Path, pages: u64) -> rusqlite::Result<()> {
    let conn = Connection::open(path)?;
    conn.execute_batch(&format!("PRAGMA incremental_vacuum({pages})"))?;
    Ok(())
}
```

- [ ] **Step 2: Create the migrations loader**

Create `src/db/migrations.rs`:

```rust
//! Load + apply toasty migrations checked into the repo.
//!
//! Toasty exposes migration generation via the CLI but the runtime
//! library applies them implicitly during `push_schema()`. For
//! production we prefer the committed-SQL path: read the migration
//! files from `toasty/migrations/`, execute each statement, stop on
//! error. Keeps the daemon from depending on the CLI.

use std::path::Path;

use rusqlite::Connection;

/// Apply every `*.sql` file in `toasty/migrations/` in filename order.
/// Safe to re-run: `CREATE TABLE IF NOT EXISTS` etc. prevent clobbering.
pub fn apply_migrations(db_path: &Path, migrations_dir: &Path) -> rusqlite::Result<()> {
    let conn = Connection::open(db_path)?;
    let mut files: Vec<std::path::PathBuf> = std::fs::read_dir(migrations_dir)
        .map_err(|e| rusqlite::Error::InvalidPath(migrations_dir.to_path_buf()))?
        .filter_map(|e| e.ok())
        .map(|e| e.path())
        .filter(|p| p.extension().and_then(|e| e.to_str()) == Some("sql"))
        .collect();
    files.sort();
    for path in files {
        let sql = std::fs::read_to_string(&path)
            .map_err(|e| rusqlite::Error::InvalidPath(path.clone()))?;
        conn.execute_batch(&sql)?;
    }
    Ok(())
}
```

(Alternative path: use toasty's `Db::push_schema()`. That creates tables based on models at runtime. Simpler but skips the committed migration files. Decision: use the committed-file path to match toasty's own todo-with-cli example convention. If `push_schema` turns out to be more reliable in practice, swap during implementation and document.)

- [ ] **Step 3: Rewrite `src/db/mod.rs`**

```rust
//! Database bootstrap and shared toasty handle.

pub mod logs;
pub mod migrations;
pub mod models;
pub mod pragma;

use std::path::{Path, PathBuf};

use toasty::Db;

use crate::errors::ExpectedError;

#[derive(Clone)]
pub struct Database {
    db: Db,
    path: PathBuf,
}

impl Database {
    pub async fn open(path: &Path) -> Result<Self, ExpectedError> {
        if let Some(parent) = path.parent() {
            std::fs::create_dir_all(parent).map_err(|e| {
                ExpectedError::database_open_failed(path.to_path_buf(), e.to_string())
            })?;
        }

        // Fresh-file pragmas via the rusqlite side-channel.
        pragma::set_auto_vacuum_incremental(path).map_err(|e| {
            ExpectedError::database_open_failed(path.to_path_buf(), e.to_string())
        })?;

        // Apply committed migrations.
        let migrations_dir = PathBuf::from("toasty/migrations");
        migrations::apply_migrations(path, &migrations_dir).map_err(|e| {
            ExpectedError::database_open_failed(path.to_path_buf(), e.to_string())
        })?;

        let url = format!("sqlite://{}", path.display());
        let db = Db::builder()
            .models(toasty::models!(
                models::Service,
                models::ServiceConfigVersion,
                models::RunningService,
                models::ServiceLog,
                models::AllocationEvent,
                models::Oneshot,
            ))
            .connect(&url)
            .await
            .map_err(|e| {
                ExpectedError::database_open_failed(path.to_path_buf(), e.to_string())
            })?;

        Ok(Self { db, path: path.to_path_buf() })
    }

    pub fn path(&self) -> &Path { &self.path }

    pub fn handle(&self) -> &Db { &self.db }

    pub fn handle_mut(&mut self) -> &mut Db { &mut self.db }

    /// Shim for `db.with_conn(|c| ...)` callers during the migration.
    /// Panics if called. Every caller must migrate to direct handle access
    /// via `db.handle()`.
    #[deprecated(note = "migrate to toasty builders via self.handle()")]
    pub fn with_conn<T>(&self, _f: impl FnOnce(&rusqlite::Connection) -> rusqlite::Result<T>) -> rusqlite::Result<T> {
        panic!("with_conn removed in toasty migration; use Database::handle()")
    }
}
```

The `with_conn` deprecation is a safety net — compile fails wherever it's still called. (Intentionally more obnoxious than necessary; forces every caller to migrate in subsequent tasks.)

Temporary: comment out or stub the existing `upsert_service` / `reparent` / `schema_version` methods on `Database`. Call sites will migrate in subsequent tasks.

Add to `src/db/mod.rs`:

```rust
impl Database {
    /// Idempotent upsert by `name`. Returns the service_id.
    pub async fn upsert_service(&mut self, name: &str, now_ms: i64) -> Result<i64, ExpectedError> {
        use models::Service;

        if let Ok(existing) = Service::get_by_name(&mut self.db, name).await {
            // Already exists. Clear tombstone if set.
            if existing.deleted_at.is_some() {
                existing.update().deleted_at(None).exec(&mut self.db).await
                    .map_err(|e| ExpectedError::database_open_failed(self.path.clone(), e.to_string()))?;
            }
            return Ok(existing.service_id);
        }
        let svc = toasty::create!(Service {
            name: name.to_string(),
            created_at: now_ms,
            deleted_at: None,
        })
        .exec(&mut self.db)
        .await
        .map_err(|e| ExpectedError::database_open_failed(self.path.clone(), e.to_string()))?;
        Ok(svc.service_id)
    }

    pub async fn reparent(&mut self, old_name: &str, new_name: &str, now_ms: i64) -> Result<(), ExpectedError> {
        use models::Service;
        let old = Service::get_by_name(&mut self.db, old_name).await.ok();
        let new = Service::get_by_name(&mut self.db, new_name).await.ok();
        match (old, new) {
            (Some(old), None) => {
                old.update().name(new_name.to_string()).exec(&mut self.db).await
                    .map_err(|e| ExpectedError::database_open_failed(self.path.clone(), e.to_string()))?;
            }
            (Some(old), Some(_)) => {
                old.update().deleted_at(Some(now_ms)).exec(&mut self.db).await
                    .map_err(|e| ExpectedError::database_open_failed(self.path.clone(), e.to_string()))?;
            }
            _ => {}
        }
        Ok(())
    }
}
```

If `Service::get_by_name` isn't auto-generated (it should be because of `#[unique]`), fall back to `Service::filter(Service::FIELDS.name().eq(name)).first().exec(...)`.

## Task 5: Migrate `running_services` (spawn + drain sites)

**Files:** `src/supervise/mod.rs`, `src/supervise/orphans.rs`.

- [ ] **Step 1: Replace INSERTs in supervisor**

In `src/supervise/mod.rs::run`, the spawn path does `db.with_conn(|c| c.execute("INSERT INTO running_services...", ...))`. Replace with:

```rust
let rs = toasty::create!(crate::db::models::RunningService {
    service_id,
    run_id,
    pid: pid as i64,
    spawned_at: chrono_like_now_ms(),
    command_line: cmdline.clone(),
    allocation: allocation_json,
    state: "starting".to_string(),
}).exec(db.handle_mut()).await;
if let Err(e) = rs {
    warn!(error = %e, "running_services insert failed");
}
```

Where `db.handle_mut()` returns `&mut toasty::Db`.

Actually — the supervisor holds `db: Database` (clone-able). `Database::handle_mut` needs `&mut Database` access. The supervisor task owns its `Database` clone; use `db.handle_mut().await.await` — no, simpler: promote `Database::handle_mut` to use interior mutability so callers don't need `&mut Database`.

Resolution: toasty's `Db` is `!Clone`. But we need it accessible from async tasks. Store `toasty::Db` inside an `Arc<Mutex<...>>` on `Database`? That hurts concurrency.

**Simplest**: store `toasty::Db` directly and make `Database` itself clone via Arc. Check if `Db` can be wrapped that way.

Actually look at the signature:
- `toasty::Db::connection()` returns `Connection` from a pool.
- Most CRUD methods take `&mut Db` or `&mut Connection`. The mutability is about exclusive access for a single operation, not about state.

Decision: make `Database` hold `Arc<tokio::sync::Mutex<Db>>`. Methods acquire the lock briefly, run the query, release. For high-throughput paths (log batcher), acquire once per batch.

Revise `Database::handle_mut` as:

```rust
pub async fn lock(&self) -> tokio::sync::MutexGuard<'_, Db> {
    self.db.lock().await
}
```

And the supervisor:

```rust
let mut db_lock = db.lock().await;
let rs = toasty::create!(crate::db::models::RunningService { ... })
    .exec(&mut *db_lock).await;
drop(db_lock);
```

Adjust throughout.

## Task 6: Migrate log batcher

**Files:** `src/db/logs.rs`.

- [ ] **Step 1: Rewrite `flush`**

Current flush prepares `INSERT INTO service_logs(...)` and calls it in a loop within a transaction. Replace with toasty's batched create:

```rust
async fn flush(db: &crate::db::Database, buffer: &mut Vec<LogLine>, seq: &mut std::collections::HashMap<(i64, i64), i64>) {
    if buffer.is_empty() { return; }
    let lines = std::mem::take(buffer);

    let mut creates = Vec::with_capacity(lines.len());
    for line in &lines {
        let counter = seq.entry((line.service_id, line.run_id)).or_insert(0);
        *counter += 1;
        creates.push(toasty::create!(crate::db::models::ServiceLog {
            service_id: line.service_id,
            run_id: line.run_id,
            seq: *counter,
            timestamp_ms: line.timestamp_ms,
            stream: line.stream.as_str().to_string(),
            line: line.line.clone(),
        }));
    }

    let mut guard = db.lock().await;
    let tx = guard.transaction().await;
    match tx {
        Ok(mut tx) => {
            for create in creates {
                if let Err(e) = create.exec(&mut tx).await {
                    warn!(error = %e, "log flush: create failed");
                }
            }
            if let Err(e) = tx.commit().await {
                warn!(error = %e, "log flush: commit failed");
            }
        }
        Err(e) => warn!(error = %e, "log flush: begin tx failed"),
    }
}
```

The batch-statement approach (`toasty::batch((a, b, c))`) is for heterogeneous queries in a round-trip. For N homogeneous creates, sequential calls inside a transaction is the idiomatic path.

## Task 7: Migrate retention

- [ ] **Step 1: Replace trim_logs_once**

Current:
```sql
DELETE FROM service_logs WHERE timestamp_ms < ?1;
-- per-service cap:
DELETE FROM service_logs WHERE rowid IN (
    SELECT rowid FROM service_logs WHERE service_id = ?1 ORDER BY timestamp_ms ASC LIMIT ?2
)
```

Replace with toasty:

```rust
pub async fn trim_logs_once(db: &crate::db::Database, now_ms: i64) -> Result<u64, toasty::Error> {
    use crate::db::models::{Service, ServiceLog};

    let seven_days_ago = now_ms - 7 * 24 * 60 * 60 * 1000;
    let per_service = 50_000_i64;

    let mut deleted = 0u64;

    let mut guard = db.lock().await;

    // 1. Delete rows older than seven days.
    let old = ServiceLog::filter(ServiceLog::FIELDS.timestamp_ms().lt(seven_days_ago))
        .exec(&mut *guard).await?;
    for row in old {
        row.delete().exec(&mut *guard).await?;
        deleted += 1;
    }

    // 2. Per-service cap. Find services with > per_service rows, delete the
    //    oldest over-cap ones.
    let services: Vec<Service> = Service::filter(Service::FIELDS.deleted_at().is_null())
        .exec(&mut *guard).await?;
    for svc in services {
        // Toasty doesn't have count() as a cheap primitive uniformly yet —
        // fetch ids sorted ASC, skip first (total - cap), delete remainder.
        let mut q = ServiceLog::filter(ServiceLog::FIELDS.service_id().eq(svc.service_id));
        q.order_by(ServiceLog::FIELDS.timestamp_ms().asc());
        let all: Vec<ServiceLog> = q.exec(&mut *guard).await?;
        let total = all.len() as i64;
        if total > per_service {
            let excess = (total - per_service) as usize;
            for row in all.into_iter().take(excess) {
                row.delete().exec(&mut *guard).await?;
                deleted += 1;
            }
        }
    }
    drop(guard);
    Ok(deleted)
}
```

The "select then delete" approach is simpler than toasty's `in_query` at the cost of more round-trips. Acceptable for retention (runs hourly).
